# 快速入门

AI Coding Stack 是一个全面的 AI 编码生态系统目录和元数据仓库。本指南将帮助你了解如何使用网站并为项目做出贡献。

## 探索技术栈

### 浏览分类

访问 [aicodingstack.io](https://aicodingstack.io) 探索不同类别的 AI 编码工具：

- **IDE** - 查找 AI 驱动的集成开发环境（VS Code、Cursor、TRAE）
- **命令行工具** - 探索 AI 编码命令行助手（Codex、Claude Code）
- **模型** - 比较用于编码的大语言模型（Kimi K2、DeepSeek V3.1、GLM 4.5、Qwen3 Coder）
- **提供商** - 查看 LLM API 提供商（DeepSeek、Moonshot、SiliconFlow、OpenRouter）

### 比较工具

使用网站并排比较不同的工具，查看它们的功能、能力和官方文档链接。

## 贡献元数据

AI Coding Stack 由社区维护。你可以通过在我们的清单文件中添加新工具或更新现有条目来做出贡献。

### 理解清单

所有工具元数据都存储在 `manifests/` 目录下的 JSON 文件中：

- `ides.json` - IDE 元数据
- `clis.json` - CLI 工具元数据
- `models.json` - LLM 模型元数据
- `providers.json` - API 提供商元数据

### 添加新工具

1. 克隆仓库
2. 在 `manifests/` 中找到适当的清单文件
3. 按照现有架构添加你的条目
4. 确保所有 URL 可访问
5. 提交拉取请求

### 清单架构示例

**终端/IDE/CLI：**
```json
{
  "name": "工具名称",
  "id": "tool-id",
  "vendor": "供应商名称",
  "description": "简短描述",
  "websiteUrl": "https://...",
  "docsUrl": "https://...",
  "latestVersion": "1.0.0"
}
```

**模型：**
```json
{
  "name": "模型名称",
  "vendor": "供应商",
  "id": "model-id",
  "size": "70B",
  "totalContext": 128000,
  "maxOutput": 4096,
  "pricing": {...},
  "urls": {...}
}
```
